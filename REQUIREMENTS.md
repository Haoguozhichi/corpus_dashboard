# 实验数据展示平台 — 需求评审说明文档

## 一、项目概述

实验数据展示平台是一个面向 AI 研究人员和工程师的 **实验管理 Web 系统**，用于统一管理、对比和展示各类 AI 实验的结果数据。

在日常 AI 研发工作中，团队会进行大量实验：模型训练对比、NL2SQL 正确率评测、NER 实体识别评测、Agent 工具调用评测等。这些实验分散在不同的脚本、日志和 Excel 表格中，缺乏统一的管理和对比视图。

本平台提供了一套完整的解决方案：**数据录入 → 统一管理 → 可视化对比 → 轨迹追溯**。

## 二、核心需求与解决方案

### 需求 1：统一管理多种类型的实验

**痛点**：训练实验有关注准确率/损失曲线，NL2SQL 评测关注 SQL 正确率，Agent 评测关注工具调用轨迹。三类实验的数据结构完全不同，无法用同一套模板管理。

**方案**：设计三种实验类型，每种有独立的数据模型和展示逻辑。

| 实验类型 | 场景 | 核心数据 |
|---------|------|---------|
| 训练实验 | 模型训练对比 | 准确率、精确率、召回率、F1、Loss/Accuracy 曲线 |
| 评测实验 | 单轮问答评测（NL2SQL、NER） | 题目→标准答案→模型回答→正确/错误 |
| Agent 评测 | 多步工具调用评测 | 题目→执行轨迹→工具调用统计→多维评分 |

### 需求 2：实验数据批量录入

**痛点**：评测结果往往有几十到上百条，逐条手动录入效率极低。

**方案**：支持 CSV 文件拖拽批量导入 + 在线逐条编辑两种方式。CSV 自动匹配已有测试用例，未匹配到的自动创建。

### 需求 3：多维度实验对比

**痛点**：需要对比不同模型/参数配置的效果，手动制表工作量大。

**方案**：
- 仪表盘表格支持分组对比，勾选两组即可进入对比分析页
- 对比页展示指标柱状图、参数差异表、训练曲线叠加、多维评分对比
- 所有列支持点击排序

### 需求 4：Agent 执行轨迹追溯

**痛点**：Agent 评测不仅需要知道"对不对"，还需要知道"每一步怎么做的"。

**方案**：评测结果支持存储完整轨迹 JSON（thought→action→observation），详情页以时间线形式展示每步执行过程，可展开/收起。

### 需求 5：灵活的指标体系

**痛点**：不同实验可能需要不同的评价指标（BLEU、ROUGE、工具准确率等），固定指标列无法满足。

**方案**：支持自定义指标和自定义参数，以动态列形式展示在仪表盘表格和详情页中。

## 三、系统架构

```
┌──────────────────────────────────────────────────────┐
│                    浏览器 (React SPA)                  │
│  Ant Design + Recharts + React Router                 │
│  http://localhost:5173                                │
└────────────────────┬─────────────────────────────────┘
                     │ /api (Vite Proxy)
┌────────────────────▼─────────────────────────────────┐
│              Express 后端 (port 3001)                  │
│  RESTful API: categories/experiments/groups/...       │
└────────────────────┬─────────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────────┐
│                JSON 文件存储                           │
│              server/data.json                         │
└──────────────────────────────────────────────────────┘
```

### 技术选型

| 层 | 技术 | 选型原因 |
|---|------|---------|
| 前端框架 | React 19 + TypeScript | 类型安全，组件化开发效率高 |
| UI 组件库 | Ant Design 6 | 企业级组件丰富（Table、Card、Statistic、Modal） |
| 图表 | Recharts 3 | 声明式 API，React 集成好，支持折线图/柱状图 |
| 路由 | React Router 7 | SPA 页面导航 |
| 构建 | Vite 8 | 秒级热更新，开发体验好 |
| 后端 | Express 4 | 轻量，适合本地/小团队使用 |
| 存储 | JSON 文件 | 零依赖，无需安装数据库，数据可随时备份 |

### 为什么不使用数据库？

- 目标用户是小团队或个人研究者，实验数据量在百条级别
- JSON 文件可以直接用文本编辑器查看和备份
- 删除 `data.json` 即可重置数据，方便测试
- 如需迁移到数据库，API 层不变，只替换存储层即可

## 四、功能清单

### 4.1 实验类别管理
- 创建/编辑/删除实验类别
- 按类别名称和描述搜索
- 卡片式展示，每类显示实验数量

### 4.2 实验管理
- 创建/编辑/删除实验（选择 training / evaluation / agent_evaluation 类型）
- 按名称、描述、类型搜索
- 卡片式展示，显示实验组数量和实验类型标签

### 4.3 实验组管理
- 创建/编辑/删除实验组
- 动态参数：每个实验组可自定义参数键值对（如 temperature、max_tokens）
- 参数展开为独立表格列
- 点击行进入详情页

### 4.4 训练指标管理
- 统一管理所有实验组的指标数值
- 自定义指标（如 BLEU、ROUGE）
- 训练曲线数据录入（逗号分隔数值序列）
- 详情页展示 Loss/Accuracy 折线图

### 4.5 评测结果管理
- 逐条录入或 CSV 批量导入
- 编辑/删除已有结果
- 自动创建不存在的测试用例
- 详情表支持关键字筛选

### 4.6 测试用例管理
- 逐条添加或 CSV 批量导入
- 可编辑表格（支持修改题目/标准答案）
- 删除测试用例时级联删除关联评测结果

### 4.7 Agent 评测
- 轨迹 JSON 录入和多维评分录入
- 自动计算工具调用次数和错误工具次数
- 详情页展开查看轨迹时间线
- 多维评分分布图表

### 4.8 对比分析
- 仪表盘勾选两个实验组进入对比
- 训练实验：指标柱状图 + 参数差异 + Loss/Accuracy 曲线叠加
- 评测实验：准确率对比 + 参数差异 + 共同用例回答对比
- Agent 评测：准确率对比 + 多维评分横向柱状图 + 参数差异

### 4.9 导航与搜索
- 面包屑导航（每级可点击跳回）
- 返回按钮（每页右上角）
- 首页/实验列表/仪表盘均支持关键字搜索

## 五、使用流程

### 场景 A：对比两个模型的训练效果

1. 创建实验类别「图像分类」
2. 创建训练实验「ResNet 对比」
3. 创建实验组「ResNet-50」和「ResNet-101」
4. 点击「管理指标」→ 填入准确率 76.1% / 77.4% 等数值
5. 在训练曲线标签页粘贴 Loss 数据
6. 勾选两个实验组 → 点击「对比选中组」
7. 查看指标柱状图对比和 Loss 曲线叠加图

### 场景 B：评测 NL2SQL 模型

1. 创建评测实验「NL2SQL 评测」
2. 点击「管理测试用例」→ CSV 上传 50 条 SQL 题目
3. 创建实验组「GPT-4o」和「GPT-3.5」
4. 进入 GPT-4o 详情 → 「管理评测结果」→ CSV 上传评测结果
5. 重复步骤 4 为 GPT-3.5 录入结果
6. 详情页查看答案对比表，错误行高亮
7. 勾选两组对比，查看共同用例的正确率差异

### 场景 C：评测 WebAgent

1. 创建 Agent 评测实验「WebAgent 评测」
2. 添加 20 个网页操作任务作为测试用例
3. 创建实验组「GPT-4o Agent」和「Claude Agent」
4. 进入详情 →「管理评测结果」→ 手动录入含轨迹的结果
5. 详情页展示：准确率、平均工具调用、错误工具次数
6. 点击「展开」查看每步轨迹时间线
7. 对比分析页查看多维评分对比

## 六、数据格式

### 评测结果字段

| 字段 | 类型 | 说明 |
|------|------|------|
| model_response | string | 模型回答 |
| is_correct | 0/1 | 是否正确 |
| score | float | 得分 (0~1) |
| runtime_ms | int | 执行耗时 |
| token_count | int | Token 消耗 |
| trajectory | JSON | Agent 执行轨迹（可选） |
| custom_scores | object | 多维评分（可选） |
